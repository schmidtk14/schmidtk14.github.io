PORTFOLIO

Simple Post-it

Github Repository Link

Simple Post-it is a website I developed to learn the fundamentals of the React Framework. It allows the user to track and store notes to aid in organization of day to day activities. React allowed for a very simple design using state-based components to create dynamic notes. These notes resize based on the number and size of each note entry. Notes can be added or removed to fit the needs of each user. Each note can have as many entries as needed and entries can be checked off or deleted with one click. Originally a firebase backend was utilized to allow for each user to store as many notes as needed, but because this is a portfolio version of the webapp, I have currently disabled the user functionality.

Creating this website presented me with many challenges. The functionality to replicate "Notes" and "NoteEntries" which dynamically adjust in size and content required a deep dive into the capabilities of the React framework. I learned to utilize state, props, callbacks, and refs to allow for a smooth user experience. Implementing the backend was a great introduction into user authorization as well as front-end to back-end interactions. Overall, this was a great project to learn from and a great introduction to the React Framework.

EventScanner

Github Repository Link

EventScanner is part of the project I used during my time working for a small ticket brokerage. It is broken down into 3 parts, Scanners, custom Datatypes, and SQL connectors. For this portfolio I have included only 2 scanners both of which use API calls to gather their information. In truth much of the data that was needed was not in an easily accessible format. Other methods for data retrieval included web scraping, image analyzing, and email parsing. At my employers request I have omitted these more involved scanners.

The scanners include the logic used to interact with the Ticketmaster and Seat Geek APIs. They perform API requests for keyword, performer, states, among others. The information returned is then parsed into the various custom datatypes. Performer, Performance, Listing, and Sale are a few of these datatypes. Once the information is parsed into their relevant datatypes it is then written into a local SQL database which houses all the information for later use. For this portfolio, the information for connecting to the local database has been overwritten with "REMOVED FOR PORTFOLIO".

Using this method, I was able to gather an incredible amount of information in relation to events in the United States and Canada. I then used SQL queries to extract events which were of high popularity. These events were then exported into weekly Excel spreadsheets for my employer to use when deciding when and how to target the events he deemed valuable enough.

I learned and refined many skills while writing these scanners. I used the Java, Python, and SQL programming languages to perform most of the work and through practice became proficient in all three. My ability to identify problems and efficiently tackle them was tested almost daily. As always, I know that I still have much to learn but the experience I gained was invaluable.

ImageAnalyzer

Github Repository Link

Image Analyzer is a simple image recognition program I originally made as a project to learn from. It can detect images within other images or on the screen of your computer. Image Analyzer can even recognize resized or discolored images with no trouble.

This project is written in Python and utilizes OpenCV's libraries to easily detect specified images. The Imutils library is used to manipulate the images to allow for easier detection for resized or discolored images. This project is written to be easily used within larger projects and no changes to the code are necessary.

ImageTextAnalyzer

Github Repository Link

Once I had finished with my ImageAnalyzer project I then decided to see what it would take to detect text in an image. With Python this turned out to be a simple endeavor. Using the Pytesseract library I was able to quickly pull text out of most images. I then added the ability to detect the text currently on the screen by using Pyautogui to take a screen shot. The method "scan_text_in_rectangle" allows a user to scan a portion of the screen by designating the top left and bottom right points of the rectangle which the user would like scanned. This class is written to be easily incorporated in a larger project without modifications.

SimpleNotes Front-end

Github Repository Link

The front-end of Simple Notes is coded with React JavaScript. Simple Notes was coded as part of the tutorial on "serverless-stack.com". This tutorial is a full stack tutorial that served as my introduction into combining a React front-end with an Amazon Web Services back-end. It allows a user to signup, login, and create/store notes to aid in organization of day to day activities. The home page displays the current user's stored notes in a list and allows a user to add new notes. Clicking an individual note will take the user to a page which allows them to edit or delete the existing note.

This is a very simple front-end which was created mainly as a means of interacting with the AWS API that was created alongside. Completing this app was a great introduction to the AWS environment. I feel I have gained a basic grasp of some of the tools that allow for user authentication and storage on a cloud-based server. I have learned to work with "user pools", "identity pools", and "s3 buckets" among other AWS tools. Overall, this was a great tutorial and I feel I have learned a lot about serverless web apps with AWS.

SimpleNotes AWS API

Github Repository Link

Simple Notes AWS API is a serverless backend coded as part of the "serverless-stack.com" tutorial. In the process of completing this tutorial I learned much about the basics of the Amazon Web Services serverless architecture and some of its services. I learned how to create and authenticate users using "identity pools" and "user pools". I also learned to create a DynamoDB on the AWS cloud service to create and store users and the notes they create. AWS made this a straightforward process and greatly simplified the steps needed to create a secure system for managing users.

AWS is a large framework; this was only an introduction into the services it offers. That said it was a great learning experience and I feel I know enough to replicate a similar back-end in future projects. I look forward to diving deeper into the capabilities of AWS and all the services it offers.
